# BIS Standards Recommendation Engine

**BIS Hackathon 2026 — Track: AI / Retrieval Augmented Generation (RAG)**

An AI-powered system that helps Indian MSEs instantly identify applicable Bureau of Indian Standards (BIS) standards from natural language product descriptions or compliance queries.

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Set your Groq API key

Get a **free** key at [console.groq.com](https://console.groq.com), then:

pass it directly as a flag: `--groq_key gsk_...`

### 3. Place the SP21 PDF

```
data/dataset.pdf   ← put the SP21 PDF here
```

The index is built automatically on first run. To rebuild manually:

```bash
python ingestion.py --pdf data/dataset.pdf --output data/chunks.json
```

### 4. Run inference

```bash
# Recommended (free Groq key, sequential, no rate-limit issues)
 py inference.py --input data/public_test_set.json --output results/output.json --llm groq --groq_key "key" 



### 5. Evaluate

```bash
python eval_script.py --results results/output.json
```

### 6. Web UI (optional)

```bash
streamlit run app.py
```

## Inference CLI Reference

```
python inference.py --input <path> --output <path> [options]

Required:
  --input   PATH        Input JSON file (list of query objects)
  --output  PATH        Output JSON file

Options:
  --llm     BACKEND     groq (default) | anthropic | ollama | none
  --groq_key KEY        Groq API key (or set GROQ_API_KEY env var)
  --groq_model MODEL    llama-3.3-70b-versatile (default, best accuracy)
                        llama-3.1-8b-instant    (faster, lower accuracy)
  --ollama_model MODEL  Ollama model name (default: llama3)
  --workers N           1 = sequential/safe (default, free Groq tier)
                        3 = concurrent (paid Groq key only)
  --top_k   N           Standards to return per query (default: 5)
  --index_dir PATH      Index directory (default: data/index)
  --pdf PATH            PDF path for auto-ingestion (default: data/dataset.pdf)
  --no_llm              Disable LLM, use pure TF-IDF retrieval
```

## Input / Output Format

**Input** (`--input`):
```json
[
  {
    "id": "Q-01",
    "query": "Which standard covers 33 Grade Ordinary Portland Cement?",
    "expected_standards": ["IS 269: 1989"]
  }
]
```

**Output** (`--output`):
```json
[
  {
    "id": "Q-01",
    "query": "Which standard covers 33 Grade Ordinary Portland Cement?",
    "retrieved_standards": ["IS 269: 1989", "IS 8112: 1989", "..."],
    "latency_seconds": 0.95,
    "expected_standards": ["IS 269: 1989"]
  }
]
```

## Evaluation Metrics

| Metric | Formula | Target |
|--------|---------|--------|
| Hit Rate @3 | ≥1 correct standard in top 3 results | > 80% |
| MRR @5 | Mean reciprocal rank of first correct result in top 5 | > 0.70 |
| Avg Latency | Average seconds per query | < 5s |

Run evaluation:
```bash
python eval_script.py --results results/output.json
```

## LLM Backend Options

| Backend | Cost | Setup | Speed | Accuracy |
|---------|------|-------|-------|----------|
| `groq` (default) | **Free** | Get key at console.groq.com | ~800ms | High |
| `anthropic` | Paid | Set `ANTHROPIC_API_KEY` | ~600ms | Highest |
| `ollama` | **Free** | Install [ollama.com](https://ollama.com) + `ollama pull llama3` | ~2-5s | Medium |
| `none` | Free | No setup | ~5ms | Medium |

## Hackathon Compliance

- `inference.py` at root with `--input` and `--output` flags
- `eval_script.py` included and unmodified
- Output JSON contains `id`, `retrieved_standards`, `latency_seconds`
- SP21 PDF is the sole source of truth — no hallucinated standards
- LLM constrained to candidate list only (cannot invent IS numbers)
- Runs on standard CPU hardware — no GPU required
